## Communication Framework

### The Anti-Freeze Protocol

**When unsure, say:**

1. "Let me think through this systematically..."
2. "I'm considering [approach A] vs [approach B]..."
3. "I'm not certain, but here's my hypothesis..."
4. "Can I verify my understanding of [constraint]?"

**Never say:**

- _[silence]_
- "I don't know" (without follow-up)
- "This is wrong" (without explaining why)

### Think-Aloud Structure

```
1. UNDERSTAND (30 sec)
   "So we need to [restate problem]..."
   "Constraints are [list]..."
   "Clarifying questions: [ask 2-3]"

2. APPROACH (2 min)
   "I'm thinking [pattern/approach]..."
   "Trade-offs: [time/space]..."
   "Edge cases to consider: [list]"

3. CODE (15-20 min)
   "I'll start with [structure]..."
   [Explain key sections while coding]
   "This handles [edge case]..."

4. VERIFY (5 min)
   "Let me trace through with [example]..."
   "Edge cases: [test]"
   "Complexity: O(?) time, O(?) space"
```

## System Design Communication Template

### Phase 1: Requirements (5 min)

**Say out loud:**

- "Let me clarify the requirements..."
- "What's the expected scale? [users, requests/sec]"
- "Any specific features prioritized?"
- "Latency vs consistency priorities?"

### Phase 2: High-Level Design (10 min)

**Draw and explain:**

- "I'll start with the core components..."
- "Client talks to [API Gateway] which..."
- "For this scale, I'm thinking [architecture]..."

### Phase 3: Deep Dive (15 min)

**Pick 2-3 components:**

- "Let's dive into [component]..."
- "I'm choosing [technology] because..."
- "Trade-off here is [A] vs [B], I prefer [choice] because..."

### Phase 4: Handle Follow-ups (10 min)

**When asked "What if [constraint]?"**

- "Great question. That would mean [analysis]..."
- "I'd adjust by [modification]..."
- "Alternative approach could be [option]..."

## Company-Specific Prep

### Google Interview Style

- **Focus:** Clean code, optimal solutions, testing
- **Communication:** Explain trade-offs deeply
- **Practice:** LeetCode Google-tagged problems

### Microsoft Interview Style

- **Focus:** Practical problem-solving, collaboration
- **Communication:** Think through alternatives
- **Practice:** System design + behavioral mix
